## 2024-05-24 - Cross-Site Scripting (XSS) in Article PDF Export
**Vulnerability:** The `generateArticleHtml` function in `src/pages/ArticleDetail.tsx` constructs an HTML string by concatenating user-provided article data (`content.text`, `content.author`, `content.src`, `content.alt`, `content.caption`, `article.title`, `article.category`, `article.author`) without proper escaping. This is then opened in a new window via `printWindow.document.write(htmlContent)`, leading to DOM-based XSS when exporting an article.
**Learning:** Generating raw HTML strings and inserting them into the DOM directly using `document.write` or `innerHTML` without escaping is inherently unsafe. Also, simply stripping tags (e.g. via some basic sanitizers) is insufficient because it leaves attributes vulnerable (attribute injection). A proper string escaper must encode `&`, `<`, `>`, `"`, and `'`.
**Prevention:** Always use a simple HTML escape function for substituting dynamic content into strings instead of introducing heavy new sanitizer dependencies unless required, and always escape both HTML body and attribute contexts to prevent attribute injection breakouts.
