## 2026-03-30 - Fix XSS Vulnerability in PDF Export
**Vulnerability:** The `generateArticleHtml` function in `ArticleDetail.tsx` generated a raw HTML string by directly interpolating dynamic user-controlled inputs (like article titles, authors, and comments) without sanitization. This raw string was then written to a new window document for PDF download, creating a severe XSS vulnerability if an attacker crafted an article with malicious scripts.
**Learning:** This existed because the application bypassed React's built-in XSS protection by generating and writing raw HTML strings manually to `window.open().document.write()`. React normally handles escaping when rendering components, but manual DOM manipulation requires explicit sanitization.
**Prevention:** Whenever generating raw HTML strings from dynamic data, especially for features like PDF exports, email templates, or direct DOM writes, always use a utility function like `escapeHtml` to sanitize the input first, preventing special characters from being interpreted as code.
