## 2025-02-14 - Admin Password Hashing and Plaintext Prevention
**Vulnerability:** The application was vulnerable to plaintext storage of the newly updated admin password if changed within the Admin interface, while simultaneously maintaining a hashing assumption for logins, potentially leading to immediate lockout or insecure credential storage.
**Learning:** The admin interface `saveProfile` function was blindly persisting the new password string from the UI directly into the database without running it through the expected `window.crypto.subtle` SHA-256 process, due to missing validation and transformation logic on that specific form submit.
**Prevention:** Always ensure that credential update pathways (like profile edits) enforce the same cryptographic transformations and prefixing rules (`$sha256$`) as the initial seeding or authentication endpoints. Ensure a fallback `if (window.crypto && window.crypto.subtle)` exists to prevent runtime crashes during hashing in non-secure contexts.
