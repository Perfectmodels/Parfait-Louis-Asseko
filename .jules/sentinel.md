## 2025-05-09 - Conditional Password Hashing to Support Legacy System
**Vulnerability:** A hardcoded admin password was found in `useRealtimeDB.tsx` and used for plaintext login verification in `Login.tsx`. This leads to a high-severity risk of unauthorized access.
**Learning:** Fixing hardcoded credentials by directly converting all plain-text passwords into hashed alternatives can introduce breaking changes where users on legacy systems or configurations are accidentally locked out due to differing password standards. In this codebase, the login flow iterates through multiple roles (Jury, Admin, Students, Staff) making the check complex. To securely convert passwords without causing lockout regressions, it's safer to conditionally hash based on expected formats (e.g. checking length to determine if it is a legacy password or a newly created SHA-256 hash string).
**Prevention:** Always verify if a password string looks like a hash (e.g., checking if it's 64 chars long) before executing authentication. New passwords from admins should be explicitly hashed using `window.crypto.subtle` before being saved to the database.
