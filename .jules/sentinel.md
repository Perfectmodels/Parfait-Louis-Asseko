## 2024-04-23 - Hardcoded Plaintext Admin Credentials Exposure
**Vulnerability:** A hardcoded plaintext fallback admin password (`admin2025`) was exposed in `src/pages/Login.tsx` and `src/hooks/useRealtimeDB.tsx`. In client-side React applications, this allows anyone inspecting the source bundle to extract the admin password and gain unauthorized access.
**Learning:** Moving from plaintext to a client-side hash comparison prevents the password from being trivially exposed. However, it requires ensuring "pass-the-hash" attacks are prevented by checking that only the admin role uses the hashed password comparison (so standard users cannot login by sending the hash string as their password) and by confirming `window.crypto.subtle` is available in the environment.
**Prevention:** Store a precomputed hash (e.g., SHA-256) instead of plaintext for client-side fallback credentials. Authenticate by securely hashing the user input using `window.crypto.subtle.digest` and comparing it strictly with the precomputed hash for the administrative role.
