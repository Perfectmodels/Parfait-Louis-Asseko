## 2025-04-09 - XSS Vulnerability in PDF Document Generation
**Vulnerability:** The `generateArticleHtml` function in `src/pages/ArticleDetail.tsx` dynamically injected user-controlled data (like `article.title`, `article.author`, and content fields) directly into a raw HTML string template via template literals. This string was then executed using `printWindow.document.write(htmlContent)`, completely bypassing React's built-in XSS protection and opening a vector for Stored XSS if article content was manipulated.
**Learning:** React only protects against XSS when rendering components within its tree. Whenever the codebase breaks out of the React lifecycle to generate raw HTML strings (such as creating new windows, sending HTML emails, or generating PDFs), all dynamic data must be manually sanitized.
**Prevention:** Always use a sanitization utility (like the newly created `escapeHtml` in `src/utils/sanitize.ts`) to escape special characters (`&`, `<`, `>`, `"`, `'`) when building raw HTML strings from dynamic data.
