## 2025-06-03 - Hash admin passwords in code and storage
**Vulnerability:** The default admin password was hardcoded in plaintext (`admin2025`) in the source code (`src/hooks/useRealtimeDB.tsx` and `src/pages/Login.tsx`), and the application updated and saved new admin passwords in plaintext within the Firebase Realtime Database.
**Learning:** Legacy authentication approaches often default to plaintext string comparisons for simplicity, creating significant risk if the database or repository is compromised. Adding cryptographic hashing requires careful handling to support graceful fallbacks for existing deployed instances (e.g., conditionally checking for a `$sha256$` prefix) to avoid breaking existing valid logins.
**Prevention:** Never hardcode passwords in the codebase. Always hash sensitive credentials (using standard algorithms like SHA-256 via `window.crypto.subtle.digest`) on the client before transmitting or saving them to the backend, especially when relying on a Realtime Database that might lack deeper backend-side hashing hooks.
