## 2025-02-28 - DOM-based XSS in PDF Generation via document.write
**Vulnerability:** The application was vulnerable to DOM-based XSS because `src/pages/ArticleDetail.tsx` generated raw HTML string templates inside `generateArticleHtml` that appended unsanitized user inputs (e.g., `article.title`, `article.author`, `content.text`) directly, which were then executed via `printWindow.document.write()`.
**Learning:** Raw string concatenations to construct HTML templates inside single-page applications bypass normal React or framework sanitization defenses, exposing critical vulnerabilities when DOM APIs like `document.write` are used. Simple data models (like `Article`) must still have string properties properly escaped in contexts lacking auto-escaping.
**Prevention:** In instances where raw HTML must be dynamically generated from untrusted data (like custom PDF downloads using `window.open`), all dynamic values—including variables and object properties—must be explicitly passed through a robust `escapeHtml` function that covers the five critical entities (`&`, `<`, `>`, `"`, `'`).
