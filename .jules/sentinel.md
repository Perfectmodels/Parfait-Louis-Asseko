## 2024-05-18 - XSS via Raw HTML String Construction
**Vulnerability:** The application was vulnerable to Cross-Site Scripting (XSS) in the article PDF generation feature. User-generated content from `src/pages/ArticleDetail.tsx` (titles, authors, and article body text) was injected directly into a raw HTML string using template literals without sanitization. An attacker could inject malicious scripts through article fields which would execute when the user clicked the PDF download button (which calls `window.open` and writes the HTML).
**Learning:** React's built-in XSS protection (which automatically escapes values rendered in JSX) does not protect code that manually concatenates strings to generate HTML to be injected or rendered by other means (like `document.write` or `dangerouslySetInnerHTML`).
**Prevention:** Always use a dedicated sanitization function (like the custom `escapeHtml` utility or a proven library like `DOMPurify`) to escape *every* dynamic variable before interpolating it into a raw HTML string. This must include both text content and HTML attributes.
