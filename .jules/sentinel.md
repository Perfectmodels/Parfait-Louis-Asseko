## 2024-03-10 - [CRITICAL] Hardcoded Admin Password Removal
**Vulnerability:** A critical security vulnerability was found where the main admin password (`'admin2025'`) was hardcoded directly into the client-side code (`src/pages/Login.tsx` and `src/hooks/useRealtimeDB.tsx`). This exposed administrative credentials to anyone inspecting the frontend bundle.
**Learning:** Hardcoding sensitive information like passwords or API keys in frontend code is an anti-pattern as Vite statically replaces them or exposes them in the build. The correct approach is to rely on environment variables (`VITE_ADMIN_PASSWORD`) and cast them as needed. This was specifically mentioned in the project constraints but not implemented correctly.
**Prevention:** Ensure that all secrets and sensitive configuration are strictly managed through `.env` files and accessed via `import.meta.env.*`. Code reviews and automated secret scanning tools should be used to prevent such hardcoded values from entering the repository.
