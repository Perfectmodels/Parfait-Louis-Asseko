## 2025-02-28 - Hardcoded Plaintext Admin Password and Insecure Password Update Logic
**Vulnerability:** The default admin password was hardcoded in plaintext (`'admin2025'`) in `src/hooks/useRealtimeDB.tsx` and `src/pages/Login.tsx`. Furthermore, when the admin updated their profile in `src/pages/Admin.tsx`, the new password was saved to the Firebase Realtime Database in plaintext. The `firestore.rules` and `database.rules.json` files currently have insecure development rules allowing global read/write access, exposing the plaintext password to anyone who can access the database.
**Learning:** Hardcoding credentials and storing user-submitted passwords in plaintext exposes the application to severe risks, particularly when database security rules are overly permissive for development purposes. Implementing client-side hashing (e.g., SHA-256) before storing passwords provides a baseline defense against credential exposure in the event of a database leak.
**Prevention:** Always use secure hashing algorithms (like SHA-256 via `window.crypto.subtle`) for storing and comparing passwords, even for default or fallback credentials. Ensure that password update forms do not expose the existing hash to the client and securely hash the new password before persisting it to the database. Avoid hardcoding plaintext secrets in the source code.
