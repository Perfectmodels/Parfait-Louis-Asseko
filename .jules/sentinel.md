## 2025-02-14 - Prevent Hardcoded Fallback Passwords
**Vulnerability:** A hardcoded plaintext admin password (`admin2025`) was present as a fallback profile in `src/hooks/useRealtimeDB.tsx` and in the authentication fallback logic of `src/pages/Login.tsx`. This exposed the credentials if the codebase or frontend bundle was accessed.
**Learning:** Even when environment variables are intended to be used or when data is usually fetched from a database, fallback values hardcoded in client-side code are fully exposed to users. Also, "pass-the-hash" attacks can occur if the client-side login code simply accepts a hash and passes it as a valid password.
**Prevention:** Instead of hardcoding plaintext fallback passwords, securely hash them (e.g., using SHA-256) beforehand and store the hash. To avoid "pass-the-hash" vulnerabilities, ensure the login validation logic is mutually exclusive by evaluating the plaintext input against a dynamically computed hash inside the client before matching against the stored hash.
