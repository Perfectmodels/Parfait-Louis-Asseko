## 2024-04-14 - Fix Hardcoded Plaintext Admin Password
**Vulnerability:** A hardcoded plaintext admin password (`admin2025`) was found directly in the client-side source code (`src/pages/Login.tsx` and `src/hooks/useRealtimeDB.tsx`), exposing administrative access to any user who inspects the client bundle. A previous fix using `VITE_ADMIN_PASSWORD` was ineffective as the plaintext variable is still shipped in the frontend bundle, demonstrating "security theater".
**Learning:** Using environment variables with the `VITE_` prefix does not secure passwords, as they are statically injected into the client codebase during build. True client-side security for hardcoded fallbacks requires storing only a cryptographic hash.
**Prevention:** Never commit plaintext passwords to the repository. When relying on client-side authentication fallbacks, store a one-way hash (e.g. SHA-256) of the password and compare the hashed user input against the stored hash. For a robust system, rely on a proper backend authentication provider.
