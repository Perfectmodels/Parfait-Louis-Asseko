## 2024-05-30 - Fix XSS Vulnerability in PDF Generation
**Vulnerability:** Cross-Site Scripting (XSS) vulnerability found in `generateArticleHtml` function within `src/pages/ArticleDetail.tsx`. Unsanitized user inputs (e.g., article title, author, content) were directly injected into an HTML string passed to `window.open().document.write()` for PDF downloading.
**Learning:** Functions generating raw HTML strings outside of React's standard rendering lifecycle (like `document.write`) bypass React's built-in XSS protections. If these strings include dynamic data fetched from a database (even if originally generated by admins), they can execute malicious scripts if the data is compromised or if there's an indirect injection vector.
**Prevention:** Always explicitly sanitize or escape any dynamic data using a utility function (like replacing `<`, `>`, `&`, `"`, `'` with their HTML entities) before interpolating it into raw HTML strings, especially when dealing with APIs that directly parse and render HTML.
