## 2025-02-14 - Fix Cross-Site Scripting (XSS) in HTML Generation for Article Downloads
**Vulnerability:** In `src/pages/ArticleDetail.tsx`, the application dynamically generated raw HTML strings containing unsanitized user inputs (`content.text`, `article.title`, `article.author`, etc.) to facilitate PDF/Image downloads via a new window (`window.open().document.write()`). This completely bypassed React's built-in XSS protections, allowing arbitrary JavaScript execution if an attacker injected malicious HTML tags (e.g., `<script>alert(1)</script>`) into an article's text, title, or image caption.
**Learning:** Generating raw HTML strings for external windows or components creates a massive security gap in React applications. React's auto-escaping is only effective when rendering JSX/TSX components to the DOM; it offers no protection when constructing raw HTML strings programmatically.
**Prevention:** Never interpolate user input directly into raw HTML strings. If manual HTML generation is absolutely necessary (e.g., for specialized print layouts or external libraries), explicitly pass all dynamic variables through a robust sanitizer or HTML entity escaping function (like the newly created `escapeHtml`) to neutralize potential XSS payloads.
