## 2025-02-14 - Replace Hardcoded Plaintext Admin Password with SHA-256 Hash
**Vulnerability:** A fallback administrator password was hardcoded in plaintext ("admin2025") in `src/hooks/useRealtimeDB.tsx` and `src/pages/Login.tsx`. This constitutes a significant security risk, as any developer with access to the source code can view the admin credentials and potentially misuse them to gain unauthorized administrative access.
**Learning:** Hardcoded credentials are a major source of vulnerabilities. Even fallback or default credentials should be stored as cryptographically secure hashes (e.g., SHA-256) instead of plaintext, and authentication logic should be updated to compare hashes instead of plaintext strings. In frontend applications without a backend layer, hashing must be performed asynchronously on the client using the `window.crypto.subtle` API.
**Prevention:** Never commit plaintext secrets, passwords, or API keys directly to the source code repository. Always pre-compute a cryptographic hash for default passwords and store the hash instead. For dynamic password updates, ensure the new password is appropriately hashed before saving to the database. Implement automated linting rules or code scanning tools to detect hardcoded secrets before they are merged.
