## 2025-02-12 - Eliminate O(N³) Bottleneck in React Render Loops via useMemo Map Lookups
**Learning:** In complex UI components (like `BeautyContest.tsx` rendering lists of Candidates x Passages x Juries), repeatedly calling `.filter()` on the entire dataset during every render phase causes exponential slow-downs (O(N³)). A key codebase anti-pattern was performing unmemoized array reductions and filtering inline inside map loops over scores array.
**Action:** Always precompute subset relationships by grouping flat arrays into `Map` objects within a `useMemo` block before the primary render loops. Extract operations like `scores.filter(s => s.candidateId === id)` into `scoresByCandidate.get(id) || []` mapping O(N) filtering to O(1) lookups and significantly improving React rendering performance for large lists.
