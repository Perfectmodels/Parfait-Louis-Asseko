## 2024-03-24 - React.memo() Performance Optimization
**Learning:** Using `React.memo()` on repeatedly rendered list components (like `ModelCard` inside `Models.tsx`) is an effective performance optimization pattern in this application to prevent unnecessary deep re-renders when parent states (like search or filter inputs) are updated. When wrapping components typed with `React.FC` in `React.memo()`, the explicit `React.FC` type annotation should be removed. `React.memo()` returns a `React.NamedExoticComponent`, which conflicts with `React.FC` and can cause TypeScript compilation and CI deployment checks (`tsc --noEmit`) to fail.
**Action:** Always wrap components rendered in large lists that don't depend on deeply changing props (e.g. they only get an ID or primitive values) or when their direct props don't change but the parent state frequently updates (e.g., search/filter changes). Drop the `React.FC` type annotation when wrapping components with `React.memo()` to prevent type conflicts.
