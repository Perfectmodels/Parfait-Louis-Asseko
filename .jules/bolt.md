## 2025-05-26 - O(N^2) Anti-Pattern in React Render Loops
**Learning:** Found a specific codebase pattern where `Array.prototype.find()` and `.filter()` were being used inside `.map()` render and array-processing loops (`BeautyContest.tsx` and `JuryContest.tsx`). This resulted in an O(N²) or even O(N³) bottleneck for resolving relationships (candidates vs. criteria vs. passages). When data scaled, React's render phase blocked the main thread significantly.
**Action:** Replace `find()` and `filter()` within render loops by computing `useMemo` Hash Maps or nested Dictionaries (e.g., `new Map()`) ahead of the render. This shifts the complexity from O(N²) down to O(N) by turning nested loop searches into O(1) property lookups. Always pre-calculate complex relationships with `useMemo` when rendering grid-like or deeply nested data in this application.
