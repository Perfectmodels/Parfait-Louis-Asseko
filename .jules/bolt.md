## 2025-02-12 - Eliminate O(N³) nested array lookups during matrix render
**Learning:** During beauty contest grid/matrix rendering (like in `AdminBeautyContest.tsx` and `JuryContest.tsx`), the `getScore` helper was using an `Array.prototype.find()` on the `scores` array. Since this helper is called repeatedly inside a heavily nested render loop (passages × candidates × juries), it created an O(N³) rendering bottleneck, significantly degrading UI performance.
**Action:** When building list-heavy or nested UI components that require frequent cross-referencing of data sets, pre-compute a lookup dictionary using `useMemo` with a `Map` (often using composite string keys like `juryId-candidateId-passageId`) to turn expensive array `.find()` calls into O(1) lookups before the render loop.
