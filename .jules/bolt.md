## 2024-05-18 - Replacing frequent array.find() and array.filter() operations with O(1) Map lookups in nested render loops
**Learning:** In list-heavy UI components, specifically inside complex tables like the scoring matrix in AdminBeautyContest where juries, candidates, and passages form a nested loop (Candidates × Passages × Juries), using .find() and .filter() inside the inner loop is extremely inefficient (O(N^3) time complexity), significantly delaying render times.
**Action:** Always precompute lookups using useMemo with a Map structure before iterating to achieve an O(1) lookup inside the render loop. I converted scores.find(...) and criteria.filter(...) to rely on scoreMap.get(...) and criteriaMap(passageId) which resolves the nested loops overhead in AdminBeautyContest.tsx and similar files.
