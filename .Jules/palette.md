## 2026-04-07 - Focus states for floating utility buttons
**Learning:** For interactive floating utility buttons, applying focus states properly is crucial for keyboard navigation visibility. Also, if the clickable element has rounded visual bounds inside, the parent interactive element (e.g. `<button>` or `<Link>`) must have the contour applied (e.g. `rounded-full`) alongside the focus ring, otherwise the focus ring outlines as a default rectangle contour wrapping a rounded visual shape, looking misaligned.
**Action:** When adding focus states (`focus-visible:ring-2`) to parent interactive elements like `<button>` or `<Link>`, ensure the border radius/shape utility classes (e.g. `rounded-full`) are also applied on the parent element itself so the focus ring correctly wraps the element visually.
